- 개인 PC가 아닌 저가의 VPS에 배포해 에이전트를 격리(sandbox)하면 안전하고 항상 온라인 상태의 24/7 에이전트를 운영할 수 있다.

## 주요 내용
- 개념 정리
  - OpenClaw 본질: LLM 자체가 아님. 여러 LLM(원격 상용 또는 로컬)을 '브레인'으로 연결해 그 브레인에 '바디'(컴퓨터, API, 메시징 채널 등)를 부여하는 오케스트레이션 계층.
  - 에이전트 기능: 브라우저 자동화, 터미널/코드 실행, 이메일 읽기/전송, 드라이브/문서 접근, 앱 배포, 스케줄·리마인더, 웹 브라우징, 주기적 리포트 등.
  - 인터페이스: 별도 대시보드 없이 평소 쓰는 채팅 앱으로 명령·상호작용(모바일 우선 UX).

- 배포 및 초기 설정(핵심 단계)
  - VPS 사용 권장: 개인 PC 대신 별도 VPS에 배포해 격리 운영(월 저렴한 비용으로 24/7 가동, 장애 시 서버 삭제로 복구).
  - 원클릭 템플릿 활용: 호스팅 제공업체의 원클릭 이미지로 Docker/의존성 설치 과정을 건너뛰고 빠르게 배포 가능. (직접 수동 설정도 가능)
  - VPS 사양 권장 예시: 2 CPU / 8GB RAM / 100GB 디스크 수준(동영상 기준 권장).
  - LLM 연동: 사용하려는 LLM 제공자의 API 키를 준비해 대시보드에 연결(여러 모델을 전환 가능).
  - 메시징 채널 연결: 모바일 채팅 앱 계정을 봇/에이전트와 페어링해 메시지로 상호작용(봇 토큰·페어링 코드 사용).
  - 이메일·깃허브·배포 플랫폼 연동: 앱 비밀번호, 액세스 토큰 등을 발급해 에이전트에 권한 부여하면 메일 스캔, PR 확인, 배포 수행 가능.

- 에이전트 구성 요소
  - soul.md (정체성/페르소나): 에이전트의 성격·행동 규칙·우선순위를 서술해 응답 스타일과 경계 설정.
  - 메모리(영구 컨텍스트): 대화와 문서를 누적해 프로젝트·선호·결정 사항을 장기 보관(시간 지남에 따라 에이전트가 더 유용해짐).
  - 스킬(플러그인): 커뮤니티 또는 사용자가 만든 기능 확장. 사용자가 직접 대화로 신규 스킬을 생성·설치 가능.
  - 모델 전략: 가벼운 모델은 일상 작업(빠르고 저렴), 무거운 모델은 복잡한 리서치/추론용으로 분리 운영.

- 실제 워크플로우 데모(핵심 사례)
  - 모바일에서 프로젝트 생성·배포: 에이전트에게 요구사항을 보내면 저장소 생성, 코드 작성·테스트·푸시, 배포까지 자동 수행.
  - 코드리뷰·PR 관리: 오픈 PR 분석, 우선순위·보안 이슈 판별, 필요시 코멘트 작성·병합 제안.
  - 이메일 인박스 스캔: 앱 비밀번호(2단계 인증 필요)로 연결해 응답 필요 메일 요약·초안 작성·자동 발송.
  - 리서치 자동화: 웹/깃허브 데이터를 수집해 비교 리포트 작성, 주기적 리서치 스케줄링.
  - 지속적 컨텍스트 활용: 대화를 통해 누적된 내용으로 '두뇌'처럼 프로젝트 전반의 상태를 요약·추적.

## 실무 포인트
- 보안·격리
  - 개인 PC에서 직접 실행하지 말 것. 별도 VPS에 배포해 에이전트를 격리 운영하면 개인 파일·비밀번호 유출 위험을 줄일 수 있다.
  - 외부 서비스 API 키, 이메일 앱 비밀번호, 배포 토큰 등 민감 정보 관리는 최소 권한 원칙으로 설정(짧은 만료·필요 시 재발급).
- 비용·API 관리
  - LLM 호출은 별도 과금(페이퍼유즈). 초기 테스트용 무료 크레딧을 쓰다 더 필요한 경우 충전 필요. 대형 모델은 비용·속도 부담이 크므로 경량 모델을 기본으로 사용.
  - 사용 초반 캐시가 없어서 API 사용량이 높게 나오지만, 시간이 지나면 캐시로 비용·호출량이 줄어듦.
- 운영·성능
  - 메모리 관리: 컨텍스트가 빠르게 쌓여 응답 지연이나 비용 증가를 초래할 수 있으므로 주기적으로 기억 항목을 검토·정리하거나 만곡(compaction) 전에 불필요한 항목을 플러시.
  - 모델 장애 대비: 상용 모델의 일시적 장애 발생 가능. 여러 제공자를 설정해 대체 모델로 전환할 수 있도록 구성.
  - 권한 실수 주의: 자동 공개 행위(트윗, 공개 이메일 등)는 신중히 제한하고, 내부 액션·외부 액션 경계를 분명히 세울 것.
- 확장성·운영 팁
  - 음성 입력(음성→텍스트) 지원으로 운전 중 위임 등 자연스러운 인터페이스 활용 가능.
  - 멀티 에이전트 운영 고려: 업무용·개인용 등 에이전트 별로 메모리·권한을 분리하면 업무별 최적화 가능.
  - 스킬을 통한 맞춤화: 반복되는 업무(이슈 요약, 주간 리포트 등)는 스킬로 구현해 자동화하면 시간이 쌓일수록 생산성 극대화.
- 한계와 전제조건
  - 초기 설정(토큰/권한 발급, 2단계 인증 등)을 직접 수행해야 하며, API 사용료·VPS 비용은 지속 발생.
  - 에이전트의 행동은 모델·설정에 따라 틀릴 수 있으므로 자동화된 변경(코드 푸시·배포 등)은 최소한의 검토 프로세스를 두는 것이 안전.
