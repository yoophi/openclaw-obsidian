## 주요 내용
- 구성요소 개요
  - 5개 모듈: BMAD 방법(BMM/BMAN), 빌더(Agent Builder), 테스트 아키텍트(Test Architect), 크리에이티브 인텔리전스(Creative Intelligence), 게임 개발 스튜디오(간단 소개).
  - 규모: 26개 에이전트, 68개 워크플로가 기본 포함.
  - 커뮤니티 기여: 소프트웨어 베테랑들이 프롬프트·워크플로를 지속 보강 중.

- BMAD 방법(개발 프로세스)
  - 그린필드: 신규 제품 중심. 4단계 흐름 — 브레인스토밍(제품 포지셔닝, 경쟁/리서치, 제품 브리프) → 설계(PRD/UX/스택·역할 정의) → 솔루션링(데이터베이스·아키텍처, 에픽·스토리·태스크 정의) → 구현 루프(스토리 생성→개발→리뷰→QA→반복).
  - 브라운필드: 기존 앱 개선·버그 수정. 간단한 변경은 Quick Spec/Quick Dev 트랙, 큰 기능은 PRD 통한 풀 프로세스 권장.
  - 업무 분해: 에픽(대단위) → 스토리(중단위) → 태스크(세부 작업). 에이전트에게 태스크 단위로 할당해 순환 개발.

- 에이전트·워크플로 동작 방식
  - 각 에이전트는 역할(비즈니스 애널리스트, 아키텍트, 개발자, QA, PM, UX 등)을 모방하는 프롬프트와 메뉴 커맨드를 가짐.
  - 워크플로 문서(마크다운/YAML 등)를 특정 명령과 연결해 필요할 때만 컨텍스트를 주입(context injection). 불필요한 전체 문서 로딩을 피해 대화가 가벼움.
  - Claude(또는 유사 LLM) 내에서 슬래시 명령으로 에이전트를 실행하면 메뉴형 명령을 통해 해당 워크플로를 불러와 실행.
  - BMAD Builder로 커스텀 에이전트를 생성·구성 가능. 도메인별 에이전트(컨텐츠, 헬스, 제안서 생성 등) 제작 지원.

- 테스트·크리에이티브 모듈
  - Test Architect: 분리된 모듈로서 자동화된 QA·테스트 설계 지원. 대형·복잡한 시스템에서 필요한 엔터프라이즈급 테스트 워크플로 제공.
  - Creative Intelligence Suite: 초기 기획·아이디어 발굴을 보조. 건너뛰기 쉬운 기초 단계(브레인스토밍, 포지셔닝 등)를 보완해 일관된 설계 기반 확보.

- 마켓플레이스(예정) 및 보안 이슈
  - 향후 BMAD 마켓플레이스에서 도메인 전문가가 만든 검증된 에이전트 공유 예정. (현재 Claude Skills처럼 방대한 수의 비검증 스킬과 차별화할 계획)
  - 외부 에이전트·스킬은 검증·보안 리스크가 있을 수 있으므로 검증된 에이전트 사용 권장.

- 설치·지원 도구
  - CLI 기반 한 명령 설치·인스톨러 제공(영상에서 시연). 공식 문서와 BMAD 도움(BMAD help) 채널로 실시간 도움 가능.

## 실무 포인트
- 문서 우선: 프로젝트 시작 전에 제품 브리프·PRD·에픽/스토리 구조를 먼저 만들면 LLM이 일관된 결정을 하게 되어 환각·중복 구현을 줄일 수 있다.
- 컨텍스트 최소화: 워크플로 파일을 명령과 연결해 필요한 문서만 주입하라. 대화 컨텍스트를 과도하게 넣으면 모델이 혼동하거나 느려질 수 있음.
- 에이전트 역할 고정: 에이전트마다 책임(예: Mary = 비즈니스 애널리스트)을 명확히 해 역할중복·지시 혼선을 방지.
- 브라운필드 전략: 작은 변경은 Quick Spec/Quick Dev로, 대형 기능은 그린필드처럼 PRD 기반 전 과정을 수행하라.
- 테스트 자동화 우선: Test Architect 모듈을 이용해 자동화 테스트·QA를 설계해두면 나중에 스케일·유지보수에 유리하다.
- 커스텀 에이전트 설계 시 보안 검토: 외부 데이터·API키 접근이 필요한 에이전트는 권한·비밀정보 관리 정책을 미리 정해둘 것.
- 커뮤니티·프롬프트 관리: 기본 에이전트 프롬프트는 커뮤니티가 다듬은 베스트 프랙티스지만, 프로젝트별로 버전 관리하고 검수할 것.
