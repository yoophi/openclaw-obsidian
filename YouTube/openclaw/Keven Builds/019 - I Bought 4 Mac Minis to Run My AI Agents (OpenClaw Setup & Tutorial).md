## 주요 내용
- 초기 브라우저/계정
  - Safari 기본 존재 → Chrome 설치 후 번(버너) 이메일로 로그인 권장(개인 정보 노출 방지).
- 원격 접속(스트리밍)
  - 원격 접속 앱(영상에서는 원격 데스크톱 클라이언트)을 각 기기에 설치해 중앙에서 스트리밍/관리.
  - 앱은 일회성 라이선스 설치 방식(구입 필요)으로 여러 대를 스트리밍 가능.
- 절전/디스플레이 설정
  - 시스템 환경설정에서 화면 잠금/절전 모두 '절대 안 함'으로 설정(암호 입력 필요).  
  - 실제 모니터 분리 상태에서도 화면이 켜져 있도록 더미 HDMI(EDID 에뮬레이터) 어댑터 사용(예: 4K 120Hz 호환 제품 권장).
- OpenClaw 설치 절차
  - 터미널에서 설치 스크립트 실행(홈브류 등 설치 포함). 설치 중 sudo 입력이 필요할 수 있고, 전체 설치는 약 5~10분 소요.
  - 설치 중 나오는 설정 질문에서 잘못 선택하면 다시 설정 복귀가 필요하므로 안내에 따라 정확히 선택(예: 첫 질문에서 'yes', 빠른 시작 선택).
- Open Router(API) 연결 및 모델 설정
  - Open Router에서 계정 생성 → API 키 발급 → OpenClaw에 키 붙여넣기. API 키는 안전하게 보관(암호화/비밀번호 관리 추천).
  - 모델 선택 권장: 초기 온보딩은 '4.6' 사용(비용·성능 균형). Opus는 비용이 높아 대체 모델(Sonnet 등) 사용을 권장. 온보딩 후 Haiku/Gemini Flash 등으로 전환 가능.
- 채널/봇 설정
  - Telegram 봇 연동: Telegram에서 BotFather를 통해 새 봇 생성 → 토큰 발급 → OpenClaw에 토큰 입력. 웹 UI와 Telegram 간 페어링 코드로 연결.
- 스킬·자동화 활용
  - 영상 제작 파이프라인(리서치→스크립트→음성→편집)을 자동화해 인력 대체 및 24시간 운영 목표.
  - 일부 기능(예: 이메일 관련 스킬)은 사용하지 않음—실제 비즈니스 목적에 맞게 필요한 스킬만 적용.

## 실무 포인트
- 보안
  - OpenClaw나 원격 접속 시 개인 이메일/민감 계정 사용 금지. API 키는 로컬 평문 저장 금지, 비밀번호 관리자 사용 권장.
- 비용 관리
  - 고성능 모델(예: 4.6)은 유용하나 API 비용이 빠르게 증가함. 온보딩에만 고성능 모델을 쓰고, 운영 시 더 저렴한 모델로 전환하는 전략 권장.
- 하드웨어/운영
  - 더미 HDMI 어댑터와 절전 해제 설정은 원격 접근을 위해 필수적. 원격 접속 클라이언트 라이선스는 한 번 구매로 여러 기기 제어 가능.
  - 여러 대를 동시 관리할 때 설치 스크립트를 복사·자동화하면 시간 절감(스크립트 실행·동일 답변 자동화).
- 설치 팁/주의
  - 설치 중 질문에 실수로 'no' 등을 누르면 재설정이 번거로움. 첫 설치는 천천히 안내 따라 진행.
  - Telegram 페어링은 때때로 페어링 코드 입력 과정이 필요하므로 모바일 Telegram 준비.
- 한계
  - 시스템 운영 비용(하드웨어 + API 사용료)이 발생하며, 초기 설정·디버깅이 필요. 완전 무인화 전까지는 모니터링 필요.
