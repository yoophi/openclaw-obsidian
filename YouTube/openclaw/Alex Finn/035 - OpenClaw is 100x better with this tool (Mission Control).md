## 핵심 요약
- OpenClaw에 'Mission Control' 대시보드를 추가하면 에이전트가 필요한 도구를 즉시 생성하고 전체 워크플로우를 관리할 수 있다. *Mission control is a custom dashboard for your OpenClaw that allows it to build any tool it needs on the fly.*
- 핵심 화면(태스크보드, 캘린더, 프로젝트, 메모, 문서, 팀, 오피스)을 통해 작업 가시성·스케줄링·기록·에이전트 조직을 한곳에서 관리한다.
- 모든 도구는 코드 작성 없이 OpenClaw에게 프롬프트로 요청해 자동 생성 가능하며, 개인 목적에 맞게 *역프롬프트(reverse prompting)*로 맞춤 확장이 중요하다.
- 대시보드를 재미있고 시각적으로 구성하면 사용 빈도와 생산성이 올라간다. *This makes it so much better and also just a lot more fun to use.*

## 주요 내용
- 기본 개념
  - Mission Control: OpenClaw 전용 커스텀 대시보드로 에이전트가 필요한 UI·도구를 실시간 생성하고 관리.
  - 무코드 방식: 간단한 프롬프트(예: "내 Mission Control을 Next.js로 만들어 로컬호스트에 호스팅해줘")만으로 템플릿과 화면을 자동 생성.
- 핵심 화면별 기능
  - 태스크보드(Task board)
    - 칸반 스타일로 각 태스크 상태 표시, 담당자(A = 사용자, H = Henry 등) 지정.
    - 왼쪽에 실시간 액티비티 피드 제공으로 에이전트 활동을 세부 확인.
    - "Heartbeat"로 주기적으로 백로그를 확인해 에이전트에게 할당된 태스크를 자율 수행.
  - 캘린더(Calendar)
    - 에이전트가 예약한 크론 작업·스케줄된 태스크를 시각화해 사전·사후 확인 가능.
    - 에이전트의 '약속했지만 실행 안 함' 문제를 발견하고 검증하는 용도.
  - 프로젝트(Project)
    - 주요 프로젝트 목록과 진행도 관리. 태스크·메모·문서와 연동해 전반적 진척 파악.
    - 역프롬프트로 즉시 진척에 도움이 되는 단기 과제를 추천받을 수 있음.
  - 메모(Memory)
    - 일별로 정리된 대화·기억을 저널처럼 조회. 장기 메모 문서도 함께 관리해 과거 대화 추적 가능.
    - 기본적으로 흩어지는 메모(raw 파일)를 보기 편한 UI로 정리.
  - 문서(Doc)
    - 에이전트가 생성한 모든 문서를 포맷별로 저장·분류·검색 가능. 초안 찾아 재활용하기 쉬움.
  - 팀(Team)
    - 에이전트(메인·서브 에이전트)의 조직도, 역할, 실행 환경(어떤 머신·엔진에서 도는지)을 한눈에 표시.
    - 최상단에 미션 스테이트먼트를 두어 모든 에이전트의 목표 정렬을 유도.
  - 오피스(Office)
    - 2D 픽셀 아트형 가시화로 각 에이전트의 현재 작업 상태를 시각적으로 보여줌. 작업 확인 및 재미 요소 제공.
- 도구 생성·확장 방법
  - 영상에서 보여준 화면은 모두 프롬프트로 생성. 사용자는 화면을 스크린샷이나 영상 링크로 제공하면 OpenClaw가 유사한 UI를 만들어 줌.
  - 초기 템플릿을 만든 뒤 역프롬프트로 개인화할 도구 목록을 계속 요청해 확장.

## 실무 포인트
- 설정 팁
  - 먼저 핵심 화면(태스크보드·캘린더·문서·메모·팀)을 만들고 이후 역프롬프트로 개인화 도구를 추가하라.
  - 태스크보드에 'heartbeat' 체크를 설정해 에이전트가 백로그를 주기적으로 확인·실행하도록 하라.
  - 미션 스테이트먼트를 정의하면 에이전트들이 우선순위를 그 목표에 맞춰 조정한다.
  - 문서·메모는 검색·카테고리화가 잘 되도록 요구 프롬프트에 명시하라(예: 형식·태그·날짜 기준).
- 주의점·전제조건
  - OpenClaw 자체의 수행 신뢰성에 의존하므로 자동화된 작업은 검토(리뷰 단계)를 반드시 포함시켜야 함.
  - 일부 실험적 기능은 아직 완성도가 낮을 수 있으니 중요한 작업은 수동 체크 포함.
  - 로컬 호스팅(Next.js 등)이나 에이전트 권한 설정 등 환경 제약이 있을 수 있음—영상에선 로컬 호스트로 호스팅을 권장.
- 한계
  - 대시보드는 에이전트가 실제로 약속을 지키는지 확인해주지만, 에이전트의 완전한 자율성과 정확성을 보장하지 않음.
  - 다른 사람의 대시보드를 그대로 복사해도 동작은 하지만 개인 워크플로우에 맞춰 커스터마이징하는 것이 더 효과적.
- 운영 팁
  - 주기적으로 역프롬프트로 "지금 우리에게 필요한 도구가 뭐냐"라고 물어 맞춤 도구를 계속 만들어라.
  - 시각적·재미 요소를 도입하면 사용 빈도와 관리 효율성이 올라간다.

## 한줄 평
OpenClaw을 실무에서 신뢰하고 활용하려면 Mission Control로 가시성·스케줄·기억·문서를 통합하고 역프롬프트로 개인화하는 것이 핵심이다.
