## 핵심 요약
- OpenClaw는 로컬에서 실행되는 자율 AI 에이전트 런타임이자 메시지 라우터로, 서브 에이전트를 이용해 작업을 자동화한다.  
- OpenCode는 터미널 기반의 오픈소스 코딩 에이전트로, 실제 코드베이스를 읽고 편집·실행하며 여러 모델을 사용할 수 있다.  
- OpenClaw(오케스트레이션) + OpenCode(실행) 조합은 명령·스케줄러 기반으로 코드 작성·디버깅·배포를 자동화하는 강력한 워크플로우를 만든다.  
- 서드파티 스킬(Claw Hub 등)은 악성 동작을 포함할 수 있으므로, 스킬 설치 전 스캐너로 검증하는 등 보안 검증이 필수다.

## 주요 내용
- 핵심 컴포넌트
  - OpenClaw: 로컬 에이전트 런타임, 메시지 라우팅, 서브 에이전트 호출 및 스킬(기능) 관리. 여러 에이전트를 조합해 작업을 분배·반복 개선 가능.
  - OpenCode: 터미널에서 동작하는 코딩 에이전트. 실제 코드베이스를 읽고 편집·추론, 파일 간 작업, 의존성 설치·테스트 실행 등 실무적 작업 수행.
  - 스킬 레지스트리(Claw Hub): 커뮤니티 제작 스킬 공유소. 편리하지만 검증이 필요.
  - 스킬 스캐너(Agent Trust Hub 등): 스킬의 악성 행위(예: 외부 설치 스크립트 실행, 민감정보 탈취 등)를 자동으로 탐지·차단하는 도구.
- 통합 워크플로우
  - 역할 분담: *OpenClaw becomes the command center* — OpenClaw이 계획·라우팅을 담당하고, *open code agent uses specialized agents trained for specific jobs.* — OpenCode가 실제 코딩·실행을 담당.
  - 모델 연동: Google Vertex(예: Gemini 3, Opus 4.6, K2.5 등) 같은 제공자를 OpenClaw/OpenCode에 연결해 고성능 모델을 사용 가능(영상에서는 Google OAuth 기반 무료 접근 시연).
  - 스케줄링·자동화: cron이나 특수 명령으로 주기적 작업 배치, 리뷰어 에이전트 대기, 오류 발생 시 자동 복구·디버깅 등 자동화 시나리오 구성.
- 설치·설정 요약 (영상에서 제시된 흐름)
  - 시스템 요구: Node 22 이상 필요.
  - OpenClaw 설치: npm 설치 명령으로 설치 후, 제공자 플러그인(예: Google 관련 플러그인) 활성화, OAuth 로그인으로 모델 접근 권한 부여.
  - OpenCode 설치: npm으로 설치 후 OpenClaw에서 제어할 수 있게 하는 스킬(예: open code controller)을 Claw Hub에서 추가.
  - 스킬 검증: 스킬 URL을 스킬 스캐너에 제출해 위험도를 확인하고, 전체 리포트 요청 가능.
- 실사용 예시
  - 텍스트 프롬프트로 CRM 대시보드 프로젝트 스캐폴딩, 의존성 설치, 개발 서버 기동까지 자동화.
  - 서브 에이전트를 배치해 지속적 디버깅·리뷰·배포 파이프라인 구성.
