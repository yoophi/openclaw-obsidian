## 주요 내용
- 핵심 개념
  - 조직(Organization): 비즈니스나 개인 삶의 영역을 논리적/물리적으로 분리해 관리하는 단위.
  - 에이전트(Agent): CEO 같은 역할의 주체. 목표를 받아 이슈를 생성·처리하고 필요 시 다른 에이전트를 생성해 위임함.
  - 이슈/프로젝트/목표: 목표(예: "매출 150% 성장") → 프로젝트 배치 → 세부 이슈로 분해되어 에이전트가 순차 처리.
  - 워크스페이스(working directory): 각 조직/프로젝트가 참조하는 로컬 폴더. 문서·스크립트·스킬을 저장해 에이전트가 사용.
  - 스킬(Skills): 시스템·사고·작업·개인 스킬 등으로 분류된 재사용 가능한 능력 집합. 에이전트가 이 스킬들을 호출해 작업 수행.
- 설치 및 실행
  - 설치 명령 한 줄로 빠르게 셋업 가능하며 브라우저 기반 UI로 접근해 조직과 에이전트를 생성·관리함.
  - 초기 구성 시 기본적으로 CEO 에이전트부터 시작하는 것이 권장됨(CEO가 구조와 권한을 설정).
- 사용 흐름(데모 사례)
  - 조직 생성 → 워크스페이스 지정 → CEO 에이전트에 목표 할당 → CEO가 이슈 분해 → 필요 시 계정/회계 등 하위 에이전트 생성 → 하위 에이전트가 실행 및 결과 보고.
  - 개인 적용 예: 건강(병원 기록, 운동 일정), 세금(서류 정리·신고 준비), 재무(계좌·이자 최적화), 경력(구직·승진 준비), 여행(조사·예약) 등 프로젝트로 분리.
- 실무 구성 포인트
  - 로컬 폴더 구조(agents, hooks, plugins, scripts, skills 등)를 미리 정리해 두면 에이전트 관리가 쉬움.
  - 스킬에 많은 시간 투자가 필요함(발표자는 수백 시간 투자해 개인 스킬 구축).
  - 에이전트가 외부 자료를 제대로 활용하려면 관련 문서·캘린더·기록을 수집·정리해 워크스페이스에 넣어야 함.
- 개발·현황
  - 오픈소스 리포지토리가 활발히 업데이트되고 있으며 빠르게 발전 중(최근 릴리스와 높은 관심도 언급).

- 데이터 준비

  - 스킬과 문서의 로컬/중앙 저장 구조를 통일하면 재사용성이 높아진다.
- 운영 팁
  - 처음에는 작은 목표(예: 주간 운동 스케줄 생성, 세금 서류 정리)로 시작해 신뢰를 쌓아가라.
  - 에이전트 생성 권한과 보고 체계(CEO가 권한 부여)를 명확히 해 자동화의 안전성을 확보하라.
  - 스킬 개발에 시간 투자할수록 에이전트 성능이 개선된다(발표자는 수백 시간 투자 경험 언급).
- 한계 및 주의사항
  - 초기 프로젝트라 문서·커뮤니티 자료가 상대적으로 적을 수 있고 지속적인 설정·관리 필요.
  - 자동화가 즉시 완전한 성과를 보장하지 않으므로 인간의 검토·승인 루프를 두는 것이 안전하다.
  - 에이전트가 제대로 동작하려면 사용자가 시스템을 설계·감독할 준비가 필요하다(구성·권한·데이터 정합성 등).
  - 에이전트가 외부 작업을 하려면 해당 작업에 필요한 입력·API·문서를 제공해야 함.
