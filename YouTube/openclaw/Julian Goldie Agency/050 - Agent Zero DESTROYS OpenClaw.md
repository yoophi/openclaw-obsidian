  - 설치는 Docker가 필요하며, 이미지 풀하고 컨테이너 실행하는 한 줄 명령으로 시작 가능.
  - 웹 인터페이스로 접근하며 실시간 진행 상황을 관찰하고 중간 개입 가능.
- 기능/역량
  - 코드 작성·실행, 소프트웨어 설치, 웹 검색·브라우징, 파일 관리, 새로운 도구 생성 등 시스템 수준 작업 수행.
  - 작업 결과를 메모리에 영구 저장해 향후 유사 문제에서 빠르게 대응(지속 메모리).
  - 메인 에이전트가 하위 에이전트를 생성해 역할 분담하고 보고받는 멀티-에이전트 워크플로우 지원.

- 필수 전제조건
  - Docker 설치가 필요하고, 로컬에서 실행하려면 충분한 CPU/RAM/스토리지 자원이 요구됨(특히 로컬 모델 사용 시).

- 설정 팁

  - 프롬프트와 메모리 구조를 먼저 이해하고 작은 워크플로우로 실험 후 점진적으로 확장할 것.
  - 하위 에이전트 분할 설계(작업 단위를 작게 나눔)는 복잡한 파이프라인에서 context 한계를 회피하는 데 유용.
- 보안·운영 주의점
  - 외부 스킬·코드 가져다 쓰는 경우 입력 검증과 코드 실행 권한을 반드시 점검할 것(서드파티 코드로 인한 데이터 노출 위험 존재).
  - 컨테이너와 호스트 간 공유 볼륨, 네트워크 접근 범위를 최소 권한 원칙으로 설정.
- 한계 및 현실적 고려
  - 메시징 기반 개인비서처럼 즉시 쓰이는 솔루션이 필요하면 추가 개발이 필요(기본 제공 아님).
  - 초기 커스터마이징에 시간 투자 필요 — 바로 쓰는 편의성보다 '통제 가능성'과 '확장성'을 택해야 하는 사용자에 적합.
  - 프로젝트 유지보수와 업데이트 모니터링은 필수(에이전트가 시스템에 많은 권한을 가짐).
