- 개인 PC에 설치하면 개인정보 노출 이슈가 있으니 가상서버(호스팅 또는 클라우드)를 권장하며, 두 가지 배포 방법(유료 원클릭 호스팅 / 무료 구글 클라우드 크레딧)을 제시한다.  
- 구글 클라우드를 이용한 설치 과정을 통해 ChatGPT(프로 플랜)를 API 비용 없이 오픈클로우와 연동해 텔레그램 등 채널로 개인 AI 비서를 운영할 수 있다.  
- 설치 과정은 Compute Engine 인스턴스 생성 → SSH 접속 → 퀵스타트 명령 실행 → 로컬에서 gcloud SDK로 로그인 링크 처리 → 텔레그램 봇 연동 순으로 진행된다.  
- 초기 온보딩에서 스킬, 외부 API 키(구글/노션/오픈AI 등)는 추후 추가 가능하며 우선 기본 텔레그램 연동으로 테스트를 권장한다.

## 주요 내용
- 오픈클로우 특징  
  - 브라우저 띄우기, 마우스 클릭 등 사용자의 행동을 직접 자동화하여 웹 기반 서비스와 상호작용 가능.  
  - 에러 발생 시 스스로 우회 경로를 찾아 작업을 완료하려는 성향이 있음.  
  - 디스코드·슬랙·텔레그램 등 다양한 인터페이스와 연동 지원.

- 배포 방식 비교  
  - 유료(간편): 호스팅 업체의 원클릭 설치(예: 월 만 원대) — 설정이 쉽고 관리 편리.  
  - 무료(학습/테스트): 구글 클라우드의 무료 300달러 크레딧 활용 — 초기 비용 절감, 다소 수동 설정 필요.  
  - 보안 관점: 개인 PC 직접 설치는 개인정보 노출 위험이 있어 별도 가상서버나 격리된 환경 권장.

- 구글 클라우드로 설치(요약된 단계)  
  1. 구글 클라우드 가입 후 무료 크레딧 활성화.  
  2. 프로젝트 생성(예: "마이 오픈클로우").  
  3. Compute Engine → 인스턴스 만들기(리전: 서울, 영역 A 등 선택).  
  4. 인스턴스의 SSH 버튼으로 접속해 터미널 창 열기.  
  5. 오픈클로우 웹사이트 퀵스타트의 리눅스용 명령 복사→터미널에 붙여넣기.  
  6. 온보딩 중 모델/프로바이더 선택: OpenAI 선택 및 관련 설정(예: 오픈AI 코덱스 등).  
  7. 웹 로그인 링크가 로컬에서 차단될 수 있으므로 로컬에서 gcloud SDK 설치 및 설정으로 인증 다리 연결(로그인 처리).  
  8. 모델 선택 후 채널 선택(예: 텔레그램).  
  9. 텔레그램 봇 생성: BotFather로 봇명 지정(마지막에 "bot"으로 끝나야 함), HTTP API 토큰 복사→터미널에 붙여넣기.  
  10. 사용자 ID는 Telegram의 UserInfo 봇 등으로 확인해 입력.  
  11. 스킬 및 외부 API 키는 건너뛰고 초기 세팅 완료 → 텔레그램에서 "안녕" 등으로 동작 확인.

- 텔레그램 연동 구체 팁  
  - BotFather로 봇 생성 시 이름 규칙(마지막에 "bot") 준수.  
  - HTTP API 토큰을 정확히 복사해 온보딩 과정에 붙여넣기.  
  - 유저 아이디가 필요하면 UserInfo 봇을 통해 확인 후 입력.
