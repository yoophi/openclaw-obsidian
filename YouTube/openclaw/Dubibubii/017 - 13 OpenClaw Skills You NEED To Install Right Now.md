## 주요 내용
- Larry (틱톡 성장 엔진)
  - 목적: 틱톡 슬라이드쇼 콘텐츠 자동 생성·업로드(초안) 및 퍼널 추적.
  - 이미지 생성: OpenAI 이미지 모델로 세로(portrait) 이미지 6장 생성. "장면 구조(scene architecture)"를 고정하고 스타일만 바꿔야 자연스러운 변환이 됨.
  - 레이아웃 규칙: 폰트 높이는 이미지의 6.5%, 텍스트는 상단에서 30% 지점(상단 10%·하단 20%는 UI에 가려짐), 문장은 4~6단어마다 줄바꿈.
  - 게시 흐름: 초안으로 올리고 사용자가 트렌딩 오디오만 추가해 게시. 음악이 도달량을 10배까지 좌우.
  - 퍼널 분석: 조회수·앱 다운로드·체험 전환·유료 전환 등 연동(RevenueCat 등)해 문제 원인(콘텐츠·온보딩·가치 제안)을 진단하고 학습.
  - 비용: 슬라이드쇼당 약 $0.50, 배치 API 사용 시 약 $0.25.
  - 요구사항: OpenAI 이미지 API 키, 틱톡 포스터·애널리틱스 연동 도구.
- Capability Evolver (자기 업그레이드)
  - 에이전트가 실패를 분석하고 스스로 코드를 수정해 개선. 두 모드: Review(수동 승인) / Mad Dog(무인 연속 업그레이드).
  - 무한 재귀 방지 안전 프로토콜 존재. Gitync 같은 백업 권장.
- QMD (토큰 절감기)
  - 로컬 인덱싱(BM25 + 벡터 검색)으로 문서에서 실제 필요한 단락만 선택해 참조.
  - 토큰 소비 최대 약 70% 감소, 문서 검색 비용 절감.
- Anti-AI Slop Humanizer
  - AI 글쓰기의 전형적 '티끌' 24가지를 감지·제거(과도한 mdash, 불필요한 흥분 표현, 규칙의 삼중 구조 등).
  - 코드 전용 버전은 과도한 주석·불필요한 방어 코드·과설계 제거.
- Exa AI Web Search (웹 검색/대안)
  - 기본적으로 에이전트는 인터넷에 접근 불가 → XAI로 웹 검색 가능(무 API키, 비용 없음) 소개.
  - 대안: Brave Browser API 이용해 무료 월 $5 크레딧으로 웹 검색 가능(가입 후 사용 한도 $5로 설정하면 실사용 과금 차단).
- GOG (Google Workspace 통합)
  - Gmail, 캘린더, Drive, 연락처, Sheets, Docs를 CLI로 통합. 아침 요약·긴급 분류·간단 회신 초안·스프레드시트 업데이트 등 자동화.
  - 설정: OAuth 자격증명 필요, 다중 계정 지원.
- 보안 경고
  - Clawhub에 수천 개(8,000+) 스킬 존재, 보안 연구진이 500개 이상의 악성 스킬(키로거·백도어 등)을 발견. OpenClaw은 VirusTotal과 제휴했지만 사용자 검증 필요.
- X(트위터) 관련 스킬들
  - Exearch Assistant: X API를 CLI로 사용. 트윗·스레드 검색, 특정 계정 모니터링, 품질 필터(최소 10 좋아요) 및 비용 투명성 제공.
  - X Impact Checker: 작성 전 트윗을 실제 추천 알고리즘 기준으로 점수화·최적화 제안.
- Bite Robot (지속 메모리)
  - 세션 간 유지되는 구조화된 컨텍스트 트리(지식베이스). BRV curate로 정보 추가, sync로 클라우드 백업.
- WhatsApp CLI
  - 에이전트가 메시지 전송·대화 검색·동기화 가능. 개인용 알림·요약 용도로만 권장하거나 아웃바운드 권한은 신중히 부여.
- Playwright (브라우저 자동화)
  - Chrome 제어: 클릭·타이핑·스크롤·스크린샷·스크래핑 등. 확장 모드(기존 로그인·쿠키 유지)와 헤드리스 모드(순수 자동화) 제공.
  - Layer 2 기능으로 이미 활성화되어 있을 수 있음.
- Mission Control
  - 이메일·캘린더·작업·메시지 등을 모아 아침 대시보드로 우선순위·집중 항목 제시. GOG 및 X 리서치와 연동 시 강력.
- 기타
  - 스킬 설치 소스: 많은 스킬이 무료로 공개(마크다운·노드 스크립트 등). 설치 전 요구 API 키·권한·비용 구조 확인 필수.

## 실무 포인트
- 보안 검증: Clawhub 스킬 설치 전 GitHub/마크다운 파일 직접 확인, 권한 최소화, VirusTotal 스캔 결과 확인 권장.
- 백업·버전관리: Capability Evolver 같은 자기수정 스킬 사용 시 Git 기반 백업(Gitync 등) 설정하고 Review 모드로 먼저 시험 운영.
- 비용 관리: API 키(이미지·브라우저·X 등) 사용 한도 설정, Brave API 무료 크레딧 활용, 배치 API로 요청 묶어 토큰 비용 절감.
- 틱톡 자동화 팁(Larry 적용 시)
  - 이미지 스타일은 '장면 구조 고정 + 스타일 변경' 방식으로 만들 것.
  - 폰트는 이미지 높이의 6.5%, 텍스트 상단 위치 30% 지점, 4~6단어마다 줄바꿈을 지켜 모바일 가독성 확보.
  - 항상 초안으로 올리고 트렌드 오디오를 수동으로 추가해 도달 최적화.
  - 퍼널(조회→다운로드→체험→유료) 지표를 연동해 무엇을 개선할지 판단하라.
- 메모리·지식베이스: Bite Robot/QMD 조합으로 반복 문서 로딩 및 토큰 낭비를 줄이고 컨텍스트를 유지하라.
- 자동 메시지 주의: WhatsApp·소셜 게시 권한은 단계적으로(자기 대상 → 소규모 그룹 → 전체) 확장하고 로그·감사 기록을 남기자.
- 자동 업그레이드 경계: Capability Evolver의 Mad Dog 모드는 강력하지만 위험(무한수정·비정상 동작) 가능성 있으니 모니터링·롤백 계획 필요.
- 테스트 습관: 모든 자동 게시물은 초기에 '드래프트'로 올려 A/B 테스팅 및 수동 검토 루틴을 두는 것이 안전.
