## 주요 내용
- 공통 개념
  - 모두 LLM을 '브레인'으로 연결하고 메시징 채널(텔레그램, 디스코드 등)을 통해 에이전트를 구동하며, 툴(브라우저, 쉘 명령, 스케줄링 등)을 호출해 작업을 자동 수행.
  - 핵심 비교 축: 비용(전력 포함), 보안(샌드박스·비밀 관리), 오픈소스 모델 지원, 설치·운영 편의성.

- OpenClaw (TypeScript / Node.js)
  - 개인용 단일 신뢰 운영자 모델(멀티테넌트 아님). 로컬 게이트웨이(기본 포트 18789)로 플러그인 채널 연결.
  - 툴: Playwright 브라우저, bash 실행, 비주얼 캔버스, 카메라/스크린 녹화, cron, webhooks, 에이전트 간 협업 등.
  - 모델: OpenAI/Anthropic/Gemini/AWS/Bedrock/Copilot/OpenRouter 등과 Alama, SGLM, VLLM 확장으로 로컬 모델 지원.
  - 보안: DM 페어링, 선택적 샌드박싱(Docker 포함), 워크스페이스 모드, exec 승인 시스템. 플러그인은 프로세스 내 실행으로 신뢰 필요.
  - 배포: interactive onboarding, systemd/launchd 서비스, Docker 지원. 비용 추적 내장.

- NanoClaw (TypeScript / Node.js, 소형)
  - 설계 철학: 최소 코드(≈4,300 라인), 컨테이너 격리를 핵심 보안 경계로 사용. 각 세션을 일시적 컨테이너로 실행.
  - 크리덴셜 프록시: 실제 API 키는 컨테이너로 들어가지 않음(플레이스홀더 사용 → 호스트 프록시가 실제 키 주입).
  - 제약: Anthropic/Claude 중심(SDK 기반). 로컬 오픈소스 모델은 1차 대상 아님.
  - 설치: ClaudeCode를 통한 AI 네이티브 설정(설정에 Claude 코드 접근 필요). 비용 추적 없음.

- ZeroClaw (Rust)
  - 100% Rust, 초경량 바이너리(≈8.8MB), 낮은 메모리(≈4MB), 빠른 시작(<10ms). 작은 하드웨어(라즈베리파이)에서 구동 가능.
  - 모듈식 확장 포인트(프로바이더, 채널, 툴, 메모리 등) 및 하드웨어 주변장치 통합(robot kit).
  - 모델/채널: 16개 프로바이더, 27개 채널, 49개 툴 포함. 로컬 모델(예: Llama) 1등 시민 지원.
  - 보안: ChaCha20-Poly1305 암호화, Landlock(커널 레벨) 등 다양한 샌드박스 옵션, 명령 허용목록·비율 제한·긴급 정지 시스템, 비용 예산 강제 적용.
  - 설치: 원라인 부트스트랩, Homebrew, 바이너리, Docker(경량 및 Debian 이미지 제공). 가장 저렴한 운영 비용.

- OpenFang (Rust, 기능 중시)
  - '에이전트 운영체제' 지향, 코드베이스 큼(133k+ 라인). 사전 구축된 자율 에이전트(=hands) 다수: 클립, 리드, 콜렉터, 예측자, 연구자, 트위터 매니저, 브라우저 자동화 등.
  - 모델/카탈로그: 27개 프로바이더, 123개 모델 카탈로그(계층화: Frontier/Smart/Fast/Local).
  - 보안: 가장 광범위한 방어층(16개). WASM 샌드박스, Merkle 해시 체인 감사, 타인트 추적, ED25519 서명, SSRF 방지, 프롬프트 인젝션 스캐너, 루프 가드 등.
  - 비용/미터링: 에이전트·모델·프로바이더별 정교한 사용량·비용 추적 및 예산 강제.
  - 설치: curl|sh 설치 스크립트, 대시보드(기본 localhost:47200), Docker 및 systemd 지원. 바이너리 ≈32MB.

- Hermes Agent (Python)
  - 고유 기능: 내장 학습 루프(사용 경험으로 스킬 생성·개선). Python 중심(3.11+), Node는 브라우저 툴용만 필요.
  - 모델: 기본 OpenRouter(200+ 모델 접근), Anthropic 및 기타 네이티브 지원. 비용 최적화: Anthropic 프롬프트 캐싱 및 컨텍스트 압축.
  - 보안/실행 백엔드: 위험명령 감지(정규식), 외부 바이너리 검증(sha256/cosign), 스킬 정적분석으로 신뢰 수준 분류. 실행환경: 로컬/컨테이너(하드닝)/SSH/서버리스 등 다양.
  - 설치: curl 인스톨러 + 인터랙티브 설정, 설정은 YAML, SQLite 사용. 스킬은 마크다운+임베디드 파이썬 코드로 저장·공유 가능(마켓플레이스 존재).

- Nanobot (Python, 경량·연구용)
  - 설계: ≈21,500 라인 Python, lm(=lightLM) 라이브러리로 100+ LLM 프로바이더 추상화.
  - 아키텍처: 이벤트 기반 메시지 버스, 에이전트 루프(약 497 라인), 세션 히스토리 JSONL, 메모리(영구/타임라인) 2층 구조.
  - 보안/제한: 위험 명령 deny 패턴, 출력/타임아웃 제한, 작업공간 제한 옵션, 디폴트로 엄격한 허용목록(deny-all by default).
  - 설치: pip/uvenv로 간단 설치, 구성 파일에 API 키 하나만 설정하면 시작 가능. Docker 지원도 제공. 비용 추적 없음.

- 비교 포인트 요약
  - 비용: ZeroClaw << OpenFang < Rust 옵션 << Python/Node 옵션(메모리 더 큼). LLM 비용(클라우드 요금)이 실제 비용의 대부분.
  - 보안: OpenFang(최상) ≈ ZeroClaw(커널 샌드박스) > NanoClaw(컨테이너 격리) > OpenClaw/Hermes/Nanobot(실용적 수준).
  - 오픈소스 모델 지원: Nanobot(가장 광범위, lightLM) 및 ZeroClaw(로컬 모델 우대) 우수. NanoClaw은 Anthropic 우선.
  - 설치 용이성: Nanobot > Hermes ≈ ZeroClaw > OpenClaw/OpenFang > NanoClaw(특이한 요구).

## 실무 포인트
- 목적에 따라 선택
  - 초저비용/항상 켜두는 개인 에이전트(라즈베리파이): ZeroClaw + 로컬 Llama 모델.
  - 엔터프라이즈급 보안·감사·완전 자율 에이전트: OpenFang.
  - 코드 가독성·연구·빠른 실험: Nanobot(간단한 에이전트 루프, lightLM 활용).
  - 에이전트가 스스로 학습·스킬 생성: Hermes Agent(학습 루프·스킬 마켓플레이스).
  - Claude(Anthropic) 중심·엄격한 격리 필요: NanoClaw(크리덴셜 프록시 + 컨테이너 세션).
  - 많은 채널·확장성: OpenClaw 또는 OpenFang(풍부한 통합 및 확장 생태계).

- 비용 관리
  - LLM 공급자 요금이 주비용. 가능한 경우 로컬 모델(VLLM, alama 등)로 전환하면 운영비 거의 0(전력비 제외).
  - OpenClaw/OpenFang은 비용 미터링 내장. Nanobot/Hermes/NanoClaw는 공급자별로 비용 관리 필요.

- 보안·운영 주의사항
  - 플러그인/확장 설치 시 신뢰 여부 확인(프로세스 내 실행 플러그인 위험).
  - 컨테이너 격리(NanoClaw)나 커널 샌드박스(ZeroClaw)의 유무와 호스트 지원 여부(Landlock 등) 확인.
  - 크리덴셜 처리 방식: 키가 에이전트 프로세스 내부로 들어가는지, 프록시나 암호화 저장을 사용하는지 확인.
  - 프롬프트 인젝션·데이터 유출 방지(특히 외부 모델 사용 시)는 필수 설정. OpenFang의 taint tracking 같은 고급 기능이 없다면 수동 검토 필요.

- 사전 조건·한계
  - NanoClaw: Claude 코드 접근 요구(초기 진입 장벽).
  - ZeroClaw: 기능이 제한적일 수 있으나 저사양 하드웨어에서 탁월.
  - Python/Node 기반 솔루션은 메모리·런타임 의존성이 더 큼(≈256MB 이상).
  - 모든 프로젝트는 오픈소스(MIT/Apache)로 전환·포크 가능하나, 운영 환경에 맞춰 커스터마이징 필요.

## 한줄 평
여섯 프로젝트는 각기 다른 우선순위(비용·보안·로컬 모델·확장성)를 목표로 설계되어 있어, 요구 사항에 맞춰 한 번의 선택으로도 충분히 실무 적용 가능하다. 어떤 목표를 중시하느냐에 따라 ZeroClaw(초저비용), OpenFang(보안·자율성), Nanobot(연구·가독성) 중 선택하면 된다.
