## 핵심 요약
- 오픈클로(OpenClaw)는 단순한 챗봇이 아니라 웹 자동화·파일·코드 실행까지 대신 해주는 실행형 AI 에이전트로, 실무 자동화에 특화되어 있다.  
- 하나의 인터페이스에서 대화·검색·코딩·브라우저 제어·파일 관리를 통합해 여러 도구를 오가며 작업할 필요를 줄여준다.  
- 예약(크론)/하트비트로 먼저 알림을 보내고 주기 작업을 실행하며, 실패 시 대체 경로를 찾아 재시도하는 복원력을 가진다.  
- 로컬 PC와 클라우드 서버 등 여러 노드를 연결해 원격 제어가 가능하지만, 그만큼 보안 리스크(프롬프트 인젝션·해킹 위험)가 커진다.  
- *말로만 어이 비서 뭐 해 줘?* — 실제로 말 한 번으로 복잡한 작업을 실행한다; *한 번 실패해도 안 멈춰요.*  

## 주요 내용
- 역할과 차별점  
  - 단순 응답형 챗봇과 달리 파일 생성, 코드 실행, 웹 크롤링/조작, 예약 작업까지 '직접 실행'한다.  
  - 여러 AI·툴을 내부적으로 호출해 필요한 기능을 자동으로 사용하므로 사용자 입장에서는 한 곳에서 처리됨.  
- 자동화·실행 기능  
  - 브라우저 자동 제어(VNC 형태 포함)로 API가 없어도 사람 대신 UI 조작(예: 유튜브 스튜디오에 로그인해 로고 변경 등) 가능.  
  - HTTP API 호출, 웹 스크래핑, 데이터 다운로드 및 분석 스크립트 작성·실행·리포트 생성까지 자동화.  
  - 크론/하트비트로 주기적 점검·리마인더·모니터링을 수행하고, 중요한 변화가 있으면 먼저 알림을 보냄.  
- 분산 노드 구조  
  - 클라우드 VPS와 집 PC 등 여러 컴퓨터(노드)를 연결해 가장 적합한 환경에서 작업을 실행하도록 구성 가능.  
  - 한 경로가 막히면 다른 노드를 경유해 문제를 해결하는 식으로 복수 경로를 활용.  
- 텔레그램 연동 및 토픽 관리  
  - 텔레그램 그룹의 토픽별 채팅을 활용하면 주제별 맥락을 분리해 기억 유지에 유리.  
  - 텔레그램을 통해 질문·명령을 주고받으며 자동화 결과를 전달받음.  
- 메모리와 지속적 맥락 유지  
  - 메모리 파일(ex. memory.md)에 투자 성향·작업 원칙 등을 저장해 AI가 지속적으로 참조하게 함.  
- 실사용 사례  
  - 영상 스크립트 작성(이전 자막/말투 분석 포함), 투자 전략 백테스트(데이터 수집→분석→리포트), 자동 로또 구매, 채널 운영(콘텐츠 제작·로고 교체), 커뮤니티 관리봇 등.  
- 검증·비용 관련  
  - 복잡한 금융·투자 분석은 AI가 잘못 계산(할루시네이션)할 수 있어, 분석 후 별도의 검증 코드로 더블체크 권장.  
  - 비용 예시: 복잡한 분석 한 번에 약 10,000원 수준, 더블체크하면 2배(약 20,000원)까지 증가 가능. 한 자동화 스크립트에서 API 호출 수천회로 몇천원 수준 발생 사례 언급.

- 검증 습관화: 돈이나 중요한 의사결정이 걸린 분석은 결과를 별도 코드로 검증(더블체크)하라. 비용이 들더라도 필수.

- 실패 복원 설계: 자동화가 에러 발생 시 대체 경로를 시도하도록 설계하면 중단 없이 작업을 완료할 가능성이 높아짐.  
- 인터페이스 선택: 텔레그램 등 메시징 플랫폼을 인터페이스로 쓰면 모바일·데스크톱에서 편하게 비서처럼 사용 가능.
