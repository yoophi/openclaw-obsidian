## 주요 내용
- 아침 브리핑
  - 매일 정해진 시각(예: 오전 6:30)에 캘린더, 이메일, 파이프라인 등을 체크해 한 페이지 요약을 지정한 채널(Telegram 예시)로 전송.
  - 요약 항목: 파이프라인 업데이트, 이번 주 클로징 예정 거래, 3일간 응답 없는 클라이언트, 캘린더 일정, 긴급 플래그된 이메일, 업계 뉴스 등.
- 이메일 트리아지
  - 수신 이메일을 자동으로 분류: 고객·제안·긴급사항을 상단, 콜드피치 등 덜 중요한 메일은 하단으로 처리.
  - 요약 제공: 오늘 이메일 수, 주의해야 할 이메일 개수 및 요점, 각 메일에 대한 추천 회신 문안 제공.
  - 보관(archiving)으로 접근성 유지하면서 받은편지함 정리.
- 영업 전 리서치(콜 전 코파일럿)
  - 특정 콜을 위해 회사 웹사이트(about, 리더십, 서비스), LinkedIn(회사 규모, 게시물, 채용공고), 최근 뉴스 등을 자동 조회해 1~2페이지 분량의 리서치 브리프 생성.
  - 브리프 내용: 회사 개요, 주요 이해관계자, 최근 동향, 예상 페인포인트, 대화 시작 문구 제안 등.
  - 팀 전체에 자동화 적용 시 다수 콜에 대해 일관된 준비 자료 제공 가능.
- 트렌드 레이더
  - X(트위터), Reddit, 보도자료, 뉴스 사이트 등 지정 소스를 스캔해 사업에 직접적인 영향을 주는 트렌드·사건을 필터링.
  - 예시 알림: 경쟁사 가격 인상(고객 확보 기회), 문제 제기하는 바이럴 스레드(콘텐츠 각도 제안), 산업 규제 변화(로드맵 영향 분석).
  - 단순 인사이트 제공을 넘어 실행 가능한 권고안도 함께 제시.
- 시그널 파인더(잠재 구매자 탐지)
  - 이상적 고객 프로필(ICP)과 관심 있는 구매 시그널을 입력하면 LinkedIn 게시물, 채용공고, 포럼, Reddit 등에서 구매 신호를 상시 탐지.
  - 발견 시 의사결정자 식별, 이메일 초안 작성, CRM에 자동 입력 등의 작업을 수행해 리드 발굴을 자동화.
  - 기존 유료 의도 데이터 도구보다 저렴한 운영 비용을 주장(작은 서버 비용).

## 실무 포인트
- 초기 설정 항목
  - 어떤 소스(캘린더/이메일/CRM/X/Reddit/뉴스)를 연동할지와 결과를 받을 채널(예: Telegram, CRM 입력 규칙)을 명확히 정할 것.
  - 아침 브리핑 시간, 이메일 우선순위 기준, 영업 리서치 템플릿, ICP와 구매 시그널 정의를 미리 설계해야 함.
- 운영·비용 관점
  - 영상 설명대로 엔지니어 팀 없이도 운용 가능하되 *"설정은 약간의 노력이 필요"*하므로 초기 튜닝 시간 배정 필요.
  - 비용은 주로 서버 운용 수준이며, 기존 의도 데이터 서비스보다 저렴하게 유사 기능을 구축할 수 있음.
- 확장성
  - 한 사람의 루틴을 자동화해 팀 전체에 확장하면 다수 콜·리드 생성 업무를 병렬로 처리 가능.
  - 자동으로 생성되는 요약·회신 초안·리서치 브리프는 팀의 생산성을 크게 높임.
- 한계와 기대치 관리
  - 완전한 원클릭 설치는 아니므로 초기에는 규칙·필터·신호 정의를 반복해서 조정해야 기대한 품질에 도달함.
