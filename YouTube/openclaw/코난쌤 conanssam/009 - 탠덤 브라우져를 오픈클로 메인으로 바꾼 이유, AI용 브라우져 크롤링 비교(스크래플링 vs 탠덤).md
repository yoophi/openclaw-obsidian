## 핵심 요약
- 텐덤(Tandem) 브라우저는 AI가 제어하도록 설계된 브라우저로, 오픈클로(OpenClo)와 연동해 에이전트가 직접 브라우저 세션을 제어할 수 있다.  
- 설치와 연동은 간단하지만 현재는 한글 깨짐, 간헐적 튕김 등 불안정성이 관찰된다.  
- 텐덤은 API 토큰과 엔드포인트를 제공해 AI가 브라우저를 직접 조작할 수 있고, 세션 연동으로 실시간 제어가 가능하다.  
- 실사용 예(네이버 부동산 크롤링)에서 텐덤은 서머리(요약)를 잘해주나 속도는 느리고 수집 개수는 제한적이었다.  
- 기존의 스크래플링(스크래플링 도구)은 패치(patch) 기능과 브라우징 모드를 혼용해 더 빠르게 많은 항목을 긁어올 수 있다.

## 주요 내용
- 설치 및 초기 동작
  - 메인 에이전트에 주소를 전달하고 설치를 클릭하면 자동으로 텐덤 브라우저가 실행된다.
  - 설치 직후 윙맨(Wingman) 패널을 통해 오픈클로 연동 패널을 확인할 수 있다.
- 주요 기능
  - 스텔스 모드, 스크립트 카드, 네트워크 실드, 모니터 활성화 등 보조 기능 제공.
  - 헤드리스(headless) 모드 토글, 스크린샷/패스워드/눈(표시) 등 UI 아이콘으로 제어 가능.
  - 텐덤 자체가 API를 제공하여 AI가 브라우저를 제어할 수 있음(토큰 기반 엔드포인트).
- 오픈클로 연동
  - 오픈클로와 세션이 연동되어 윙맨에서 내린 명령이 텐덤 브라우저에서 실행된다.
  - 작업 결과 파일은 오픈클로 워크스페이스 아래에 저장되는 구조.
- 크롤링 비교(예: 네이버 부동산)
  - 텐덤: 브라우저 직접 스크롤링하며 매물 확인, 요약 기능이 좋음. 다만 속도는 느리고 수집 개수는 제한적(예: 60개 정도까지 수집된 사례).
  - 스크래플링: 패치 기능으로 빠르게 데이터 수집 가능. SPA(single-page application)나 패치로 못 긁는 경우에는 브라우징 모드로 보완.
  - 텐덤 결과는 정리(최저가·최고가 등)를 잘해주지만 일부 가격 값이 이상하게 추출되는 등 정확성 검증 필요.
- 안정성 및 한글 처리
  - 한글이 깨지는 부분, 앱이 튕기는 현상 등 불안정한 모습이 보고됨.

## 실무 포인트
- 속도 최적화: 가능한 경우 스크래플링의 패치 기능을 먼저 사용하고, 패치로 불가능한 SPA 등은 브라우징 모드(또는 텐덤 브라우저)를 사용하라.
- 정확성 검증: 텐덤의 자동 요약은 유용하지만 추출된 수치(가격·면적 등)는 반드시 교차검증하라.
- 안정성 대비: 현재 버전에서 튕김과 한글 깨짐이 있으므로 장기 대량 작업에는 안정화된 도구(또는 재시도 로직)를 마련하라.
- 세션·토큰 관리: 텐덤 연동은 API 토큰과 세션 기반이므로 토큰 관리 및 권한 설정을 사전에 확인하라.
- 파일 위치 확인: 결과 파일은 오픈클로 워크스페이스 아래에 저장되니 저장 경로를 미리 파악해 두면 작업 흐름이 편해진다.
- 용도 구분: 텐덤은 AI와의 협업(세션 제어·요약)에 강점, 전통적 대량 크롤링은 스크래플링(패치) 방식이 더 효율적일 가능성이 높다.
