## OpenClaw: deploy a 24/7 AI agent on a VPS

This video shows OpenClaw as an orchestration layer that turns an LLM into an autonomous agent with full, sandboxed computer access, persistent memory, and a text interface. The creator demos a one‑click VPS deployment (recommended: 2 CPU cores, 8 GB RAM, 100 GB disk, large bandwidth) and connects the agent to a messaging app so you can “text it like a coworker.” Live examples include: building and deploying a small dashboard from a phone, reviewing and merging PRs, scanning and fixing a browser cookie bug found via email, scheduled research reports, and creating custom “skills” (plugins) the agent writes itself and runs on a schedule. The tutorial emphasizes safety by running the agent on an isolated VPS, explains API-key setup to plug in your preferred LLM, and shows how a simple soul.md shapes personality and behavior so the agent acts like a teammate.

Practical takeaways and limits: the full deploy can be done in under 20 minutes; the agent will initially use more API credits but caching reduces costs over time; expect occasional model rate limits and the need to top up provider credits for heavy models. Memory management is important—periodically review and prune stored context before automatic compaction—and you can mix models (lightweight for day‑to‑day, heavy models for deep work), enable voice input, or run multiple agents for different roles. The presenter backs claims with concrete demos (live code push, PR comments, PR merges, and a working morning email briefing) and shows how simple prompts produce persistent automations. *Open Claw is not an AI model. It's not a chatbot.* *You text it like a coworker.*
