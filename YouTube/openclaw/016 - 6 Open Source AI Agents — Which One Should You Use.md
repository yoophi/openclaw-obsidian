## Which open-source AI agent should you use?

This clip compares six open-source personal agent projects—OpenClaw, NanoClaw, ZeroClaw, OpenFang, Hermes Agent, and Nanobot—and explains their tradeoffs in cost, security, model support, and setup. *"There are now six serious open-source projects competing to be the brain of your personal AI assistant"* runs the thesis; the reviewer then profiles each: OpenClaw (TypeScript) is the original, single-operator gateway with ~40 extensions and broad provider support; NanoClaw (TypeScript, ~4,300 lines) is minimalist and enforces container isolation plus a host credential proxy; ZeroClaw (Rust) is ultra‑small (8.8 MB binary, under 5 MB memory, runs on a $10 Raspberry Pi) and supports 16 providers with kernel‑level sandboxing; OpenFang (Rust, ~133k lines) behaves like an “agent OS” with seven prebuilt autonomous hands, 27 providers, a 123-model catalog, and 16 distinct security layers; Hermes Agent (Python) emphasizes a built‑in learning loop and skills marketplace while using OpenRouter by default; Nanobot (Python, ~21.5k lines) is research‑friendly, uses lightLM to expose 100+ providers, and is the simplest to install. *"ZeroClaw's tagline is zero compromise 100% rust 100% agnostic."*

Recommendation highlights: pick ZeroClaw for the lowest running cost and tiny footprint; OpenFang if enterprise‑grade security and ready autonomous agents matter; Nanobot if you want the easiest setup and readable code for hacking or research; Hermes if you want a self‑improving skill loop; NanoClaw if you’re committed to Claude and want paranoid container isolation; OpenClaw if you need many channel integrations and a mature extension ecosystem. All six are open-source (MIT or MIT/Apache), require you to pay for whatever LLM provider you choose, and offer migration paths between projects in some cases.
