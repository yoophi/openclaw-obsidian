## 핵심 요약
- OpenClaw는 자체 컴퓨터에서 백그라운드로 동작하며 앱을 조작하고 반복 작업을 수행하는 ‘행동하는’ AI 에이전트다.
- 설치는 한 줄 명령으로 터미널에 실행하면 되고, 전용 기기(여분 노트북/맥 미니)나 클라우드 VPS에 두는 것이 권장된다.
- 모델 연결 방식(API 키 vs OOTH)에 따라 비용 구조가 완전히 달라지므로 설정 시 주의가 필요하다.
- 에이전트는 여러 에이전트·서브에이전트, 스킬, 플러그인, 외부 서비스(MCP) 연결으로 확장 가능하며 보안 권한 관리를 반드시 설정해야 한다.
- *OpenClaw is sort of like an employee who never clocks out.*  
  *Your OpenClaw agent basically has one brain but many ears and many ways to communicate with it.*

## 주요 내용
- 설치 및 실행
  - openclaw.ai에 있는 one-liner 명령을 터미널에 붙여넣어 설치.
  - OpenClaw는 설치한 기기에서 백그라운드 프로세스로 동작하며 파일을 생성·편집·삭제할 수 있음.
  - 권장 환경: 여분의 컴퓨터(또는 구매한 맥 미니) 또는 VPS(예: Hner, Digital Ocean, Hostinger)로 항상 온라인 유지.
- 연결 방식(비용/정책)
  - API 키: 사용량 기반 과금(무제한 비용 가능), 주로 개발자용.
  - OOTH(영상에서 사용한 표기, open authorization): 구독형 월정액(예: ChatGPT Plus)으로 비용 예측 가능. OpenAI는 허용 선언, Entropic은 불확실/회색지대, Google Gmail 계정은 차단 사례 보고.
- 핵심 구조와 동작 원리
  - 에이전틱 루프: 에이전트가 목표를 받아 스스로 계획을 세우고 도구 호출·결과 판독·추가 액션을 반복해 작업을 완수함(한 턴에서 여러 액션 수행).
  - 게이트웨이: 설치된 엔진(스위치보드)으로 채널(WhatsApp, Telegram, Slack 등)과 브라우저 인터페이스를 통해 메시지를 라우팅.
  - 채널: 에이전트와 사용자가 소통하는 경로, 한번 설정하면 동일한 방식으로 메시지 주고받기 가능.
- 작업単위와 저장소
  - 워크스페이스: OpenClaw가 지시·메모리·설정을 저장하는 폴더(기본은 OpenClaw/agents 등). 모든 핵심 설정은 일반 텍스트 Markdown 파일로 존재.
  - 핵심 마크다운 파일들
    - soul.md: 에이전트 성격·톤 설정(정중·간결 등).
    - identity.md: 이름·이모지 등 짧은 식별 정보.
    - agents.md: 운영 매뉴얼(우선순위·규칙·금지사항 등).
    - user.md: 사용자 정보(이름, 시간대, 선호 등).
    - tools.md: 외부 도구 사용법 노트(연결 방법·문제 해결 팁).
    - memory (daily notes, memory.md): 일일 로그와 장기 기억(자동으로 중요한 것만 요약 저장).
    - heartbeat.md: 기본 30분 주기의 점검 체크리스트(주기적 모니터링 내용).
    - cron jobs: 특정 시각에 정확히 실행되는 예약 작업(정시 알림·정기 보고 등).
- 컨텍스트·비용·모델 선택
  - 컨텍스트 윈도우(모델별 토큰 한계)를 초과하면 컨텍스트 엔진이 오래된 내용을 요약·압축.
  - 모든 마크다운 파일은 각 턴에 다시 주입되므로 파일 크기가 크면 매 메시지마다 토큰 비용 증가.
  - 모델 비종속적(model-agnostic): Claude, OpenAI, 지역 로컬 모델(오픈 소스) 등 필요에 따라 사용 가능—작업별로 모델을 분리해 비용/성능 최적화 가능.
- 확장 요소
  - 스킬(skill.md): 특정 작업용 플레이북(프롬프트·참고자료 포함). 에이전트별로 할당 가능.
  - 플러그인: 게이트웨이 내부에서 동작하는 코드 수준 확장(Typescript/JS 등). 새로운 채널 추가, 컨텍스트 엔진 교체 등 고급 기능 제공.
  - MCP 서버(Model Context Protocol): Google Calendar, GitHub 등 외부 서비스와의 연결 어댑터로, 에이전트가 해당 서비스에서 직접 읽기/쓰기 가능.
  - 노드(nodes): 다른 기기들을 에이전트가 제어·접근하게 하는 확장(스마트 글라스, 태블릿 등 실험적 사례).
- 보안·권한
  - openclaw.json 등 파일로 에이전트의 접근 권한을 명시적으로 허용/거부 가능(브라우저 사용 금지 등).
  - 민감한 메모리는 게이트웨이 UI에서 기본 블러 처리. 파일 접근은 로컬에서 직접 확인 가능.
  - 권장: 정기적인 보안 감사 cron 작업 설정.

## 실무 포인트
- 설치 장소 선택: 개인 주요 기기 대신 별도 전용 기기 또는 VPS에 설치해 작업 영역 분리(데이터·권한 안전).
- 비용 관리: 초보자는 OOTH(구독형)으로 시작해 월 비용을 고정하는 것이 안전. API 키는 고비용 리스크 존재.
- Heartbeat·Cron 설정 주의:
  - Heartbeat는 기본 30분 주기이므로 체크리스트를 최소화해 반복 토큰 비용을 절감.
  - 정시 작업은 cron으로 처리해 불필요한 반복 호출을 줄이기.
- 컨텍스트 최적화: 핵심 마크다운 파일(agents.md, soul.md 등)을 필요할 때만 불러오거나 요약해 매 턴 토큰 부담을 줄이기.
- 스킬·플러그인 관리: 커뮤니티 스킬은 검증되지 않은 경우가 많으니 신뢰 가능한 출처에서 받거나 자체 제작 권장. 플러그인은 코드 접근 권한을 요구하므로 설치 전 기능·보안 영향 검토 필요.
- 권한 및 보안 정책: openclaw.json로 구체적 권한 제한(예: 웹 브라우저 비허용) 설정, 민감한 데이터는 메모리 설정에서 제외하거나 암호화 대비.
- 로그·메모리 관리: daily logs와 memory.md에 민감 정보가 저장될 수 있으니 정기 점검·백업·삭제 정책 수립.
- 모델 배치 전략: 비용이 민감한 작업(간단한 자동화)은 로컬 오픈소스 모델, 복잡한 추론은 고성능 상용 모델로 분리해 사용.
