## 한글 요약
요약
이 영상은 Figma 기반 디자인 시스템(토큰·컴포넌트)을 클로드 코드(Claude Code)와 Figma MCP를 연결해 ‘바이브 코딩’ 방식으로 랜딩 페이지를 자동화·구성하는 과정을 단계별로 보여줍니다. 핵심 주장은 비개발자라도 디자인 시스템을 먼저 정의해 토큰과 컴포넌트를 코드화하면 화면 일관성·유지보수·비용 측면에서 큰 이득이 난다는 점이며, 이를 위해 *베리어블(디자인 토큰), 스타일, 컴포넌트·프로퍼티* 개념을 먼저 정리해야 한다는 점입니다. *"디자인 시스템 기반으로 코딩해서 화면을 잡으시면 일관되고 의도된 디자인 잡으실 수 있고요."*

데모에서 저자는 Figma의 베리어블을 AI로 추출해 컬러·넘버 토큰을 자동 등록하는 과정, Dev Mode MCP와 Talk-to-Figma MCP의 차이(디자인→코드 변환은 Dev Mode MCP 권장, 파일 내 주석·수정 자동화는 Talk-to-Figma MCP 유용)를 설명합니다. 실제로 프롬프트 한 번으로 97개 컬러 베리어블과 81개 넘버 베리어블을 추출해 JS 토큰 파일과 컴포넌트 구조를 생성하는 예시를 보여주며, 워크플로는 (1) 토큰 제작(베리어블) → (2) 컴포넌트에 적용 → (3) 컴파운드/템플릿 조합 → (4) Claude Code에게 MCP로 읽혀 랜딩 페이지 생성 순입니다. 실무 팁으로는 유료 기능(Dev Mode 등)이 필요하고 컨텍스트(맥락) 용량 관리(콤팩트 명령 등)와 결과 미세조정이 필요하다고 언급합니다. *"베리어블은 디자인 토큰이라는 것을 구현하는 피그마 내의 기능입니다."*
