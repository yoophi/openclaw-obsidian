## 한글 요약
## Quick take
This video demos a new, free open-source GUI that manages coding agents by using the user’s existing model subscriptions rather than replacing them. The presenter shows how the app discovers a repo, builds a plan, streams tool calls, and runs multiple agents in parallel while remaining responsive; *"It's a fully open- source agent manager that is completely free."* He highlights UI conveniences like git work-tree support, autogenerated commit messages with a one-click push+PR flow, a per-turn diff view (split or unified), an integrated terminal, quick-action buttons to run project commands, and the ability to paste a path to add a project.

## Strengths and gaps
Performance and multi-backend support are the main selling points: the app loads long threads instantly and didn’t lag with multiple agents during the demo — *"I haven't experienced that once."* The presenter notes planned support for additional coding backends (one is pending provider clarification) and that different model families work better for different tasks, so routing to the right backend matters. Missing or requested features include better “skill” management, a headless/cloud-hostable mode to manage remote agents, a visible list of running terminals, a collapsible sidebar, and easier thread renaming; because the code is open source, the presenter also shows simple local edits he made to add double-click rename, an “open in terminal” menu item, running-terminal visibility, and a sidebar toggle.
