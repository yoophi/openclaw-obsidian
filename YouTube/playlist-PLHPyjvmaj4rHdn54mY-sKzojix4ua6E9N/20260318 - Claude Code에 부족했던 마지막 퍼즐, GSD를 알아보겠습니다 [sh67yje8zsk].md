## 한글 요약
GSD(프레임워크) 요약
영상의 핵심 주장: GSD는 요구사항이 자주 바뀌고 실험이 많은 MVP형 프로젝트에 적합한 개발 프레임워크로, 과도한 사전 설계 대신 짧고 집중된 문서와 반복적 실험으로 빠르게 핵심 기능을 검증한다. *"GSD는 요구사항이 자주 바뀌거나 실험이 많은 MVP형 프로젝트에 잘 맞습니다."* 저자는 GSD가 짧은 프로젝트 MD·컨텍스트 파일로 에이전트가 핵심에서 벗어나지 않게 유지하고, 여러 섭(서브) 에이전트를 병렬로 돌려 연구를 수행하며 각 작업에 맞는 모델을 연결해 비용을 통제한다고 설명합니다.

구체적 근거: 설치는 프로젝트 단위로 적용되며, 프롬프트는 XML 구조로 되어 있어 특정 모델(예: 클로드 계열)에 최적화된다. 워크플로는 초기 질문→짧은 프로젝트 문서 생성→병렬 리서치→요구사항 정제→계획 작성→검증 에이전트의 적대적 리뷰→웨이브(병렬 작업) 실행→내부 자동화·테스트·커밋 흐름으로 이어진다. 제작 중 1차 플레이홀더 형태의 데모를 빠르게 만들고 이후 기능별로 완성해 가며, 한 이터레이션에 약 138,000토큰이 소모되었고 대형 컨텍스트 윈도우를 활용하면 더 여유롭다고 보고했습니다. 단점·권장 상황: 안전성·회수 불가능한 비용이 큰 시스템이나 이미 요구사항이 확정된 프로젝트에는 더 엄격한 프레임워크(BMAD/슈퍼파워즈류)가 낫고, 단순 앱에는 GSD가 과할 수 있다. 또한 외부 근거(문서·검색 도구)를 잘 연결해 연구 신뢰도를 높일 것을 권장합니다.
