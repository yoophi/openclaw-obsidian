핵심 실무 요점: 하네스 엔지니어는 여러 세션과 에이전트에 걸친 읽기 쉬운 환경을 설계하고 검증·피드백 루프를 짧게 만들어 모델이 스스로 검증하게 하며, 모델이 잘 다루는 범용 도구를 신뢰하는 쪽으로 시스템을 단순화해야 한다고 정리합니다. 구체적으로는 초기화 스크립트와 프로그레스 파일로 작업 상태를 명시하고 설명적인 Git 커밋과 단위·엔드투엔드 테스트(브라우저 프로토콜 스냅샷 등)로 완료를 검증하며, 문서 목차(아키텍처·스펙·DB·실행계획 등)로 컨텍스트를 분해해 각 세션이 관련 정보를 빠르게 불러오게 만드는 방식이 권장됩니다. 또한 전문 도구를 잔뜩 복잡하게 붙이기보다 단일 배치·범용 도구로 모델의 자연스러운 동작을 끌어내는 것이 실험적으로 더 효과적이었다고 결론 냅니다. *"오픈 클론은 항상 켜져 있고 능동적입니다."*
