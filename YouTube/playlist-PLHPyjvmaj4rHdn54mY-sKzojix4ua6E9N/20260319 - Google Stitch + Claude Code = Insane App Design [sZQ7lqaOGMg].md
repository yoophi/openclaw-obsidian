## 한글 요약
Google Stitch + Claude Code — core idea and features
The video shows a new Google Stitch release that embeds a design-systems-first workflow into the design-to-code loop: an AI-native canvas (accepts images, code and PRDs), a smarter design agent, voice input, instant prototypes and a formalized design system file called design.md. Stitch generates a markdown design system that documents typography, color scales, component rules and dos/don’ts, and exposes the visual frames and HTML/CSS via a Stitch MCP so coding agents can consume those specs. The presenter calls out that *“design.md really represents a step change”* and demonstrates you can *“create incredible designs just from screenshots.”*

## Workflow demo, results and limits
In the demo the creator dropped screenshots, instructed Stitch to redesign them (mobile + desktop), then edited the generated Obsidian Script Editorial design system (fonts, secondary/tertiary palettes, component examples) and copied design.md into a project. They used Claude Code to plan and implement the redesign, installed the Stitch MCP (requires a Stitch API key) so Claude Code could fetch the Stitch frame HTML/CSS, and then updated the app layout and components across several phases. Results: layouts, spacing and some interactivity (hover effects) translated well, but fonts and exact color matches weren’t perfect—design.md gives a solid, editable starting point rather than a one-click perfect conversion. The workflow raises the baseline quality of AI-generated apps and makes maintaining consistent styles in code tractable, while requiring manual checks to align features and polish typography/colors.
