## 한글 요약
Agent SDK demo: live voice-driven coding

The creator demos an AI "pool trainer" using a voice-cloned pool pro to narrate shot routes while an agent SDK connects that voice agent to a background coding agent. In the clip the voice agent listens to a spoken feature request (a “route recall” command), sends an SDK call to a coding agent, and the coding agent edits the running app so the projector overlay can later recall route one on command — all without leaving the table. *This is freaking so cool.* *That is an actual coding integration that was initiated by my voice.*

Technically, the voice agent issues a change request to an automated coding agent, which updates code (the presenter mentions restarting a Python script) and streams the SDK output back to a tabletop projector for live verification; the presenter then tests the new feature on the spot. The demo also shows the voice agent controlling studio lights as an extra example. A practical caveat noted: using agent-style API calls can be costlier and slower than routine model calls, so the presenter uses a different runtime to reduce latency and expense. If you want to see a working end-to-end example of voice-driven agents editing and redeploying code live, this clip demonstrates that workflow in action.
