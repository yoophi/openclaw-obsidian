## 한글 요약
핵심 요약
AI로 코드를 만들 때 팀 간 성패를 가르는 한 가지는 ‘코드 구조’입니다. 발표자는 도메인 주도 설계(DDD)로 비즈니스 개념(유저, 결제, 주문 등)을 코드 폴더와 이름에 그대로 반영하면 AI가 기존 패턴을 인식해 새 기능을 거의 완벽하게 복제하고 유지보수가 쉬워진다고 주장합니다. *"AI한테 백엔드 만들어 달라고 했더니 진짜로 만들어 줬어요."*는 경험적 출발점으로, 구조 없이 흩어진 코드는 AI가 매번 다르게 생성해 나중에 고치기 어렵게 만든다고 설명합니다.

구체적으로 DDD는 각 도메인에 대해(1) 데이터 정의(파일), (2) 저장/조회 책임, (3) 실제 비즈니스 규칙을 분리해 둡니다. 코드 이름을 현장 용어와 일치시키면 AI가 수정할 위치를 바로 찾고 환각(추측 생성)을 줄입니다. 일부 AI 플랫폼의 훅/트리거로 도메인별 파일만 먼저 읽게 설정하면 AI가 맥락을 파악해 일관된 스타일로 코드를 생성합니다. 발표자는 첫 도메인만은 수작업으로 만들어 AI가 판단할 기준을 갖게 하라고 권하며 핵심은 네 가지 요점으로 정리됩니다: 틀(패턴)이 있으면 AI가 이를 복사해 일관된 코드를 만든다; 도메인 단위 분리는 AI의 집중과 품질을 높인다; 비즈니스 용어와 코드 이름을 일치시키면 환각이 줄어든다; 도메인 트리거를 달면 AI가 관련 코드를 자동으로 읽어 맥락을 이해한다. *"비즈니스에서 쓰는 말이랑 코드 이름을 맞추는 거예요."*
