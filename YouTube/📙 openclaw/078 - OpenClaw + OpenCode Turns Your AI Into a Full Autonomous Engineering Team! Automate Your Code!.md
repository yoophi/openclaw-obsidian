## Quick summary — OpenClaw + OpenCode workflow, risks, and setup

The video shows how to combine OpenClaw (a self‑hosted autonomous agent runtime and message router) with OpenCode (an open‑source terminal coding agent) so OpenClaw plans and routes tasks while OpenCode reads, edits, and executes real code across files. *Essentially, OpenClaw becomes the command center with this workflow* and can dispatch OpenCode sub‑agents to scaffold projects, run continuous debugging, install dependencies, and iterate until the output is functional. The presenter demos OpenClaw building full‑stack apps from tweets, deploying multiple sub‑agents to organize files locally, and creating a working CRM dashboard using presets and model access.

The video stresses a major safety concern: many community skills in registries (Claw Hub) can be malicious and thousands of exposed instances exist on default ports. The author uses a skill scanner called Agent Trust Hub to vet skills before adding them; the scanner flagged a skill that would have downloaded and run an external installation script (malware risk). The chosen safe skill for the demo is "open code controller," which lets OpenClaw control OpenCode via slash commands, manage multiple sessions, schedule cron jobs, and route tasks like test runs, API checks, scaffolding, and code fixes. *Open code is going to handle and execute it.*

Practical setup and capabilities covered: Node 22+, npm install for OpenClaw and OpenCode, run OpenClaw onboarding/daemon, enable a model provider (the video shows enabling a Google provider via OAuth so higher‑tier models are usable), install the open code controller skill from Claw Hub, and always scan submitted skill URLs in Agent Trust Hub before adding. Once installed, you can schedule agents, run reviewer agents on production builds, have OpenClaw orchestrate complex multi‑skill workflows, and let OpenCode implement and refine code autonomously—examples include spinning up a Node/React scaffold, installing deps, starting a dev server, and iterating UI and DB/auth features. The presenter emphasizes verifying each skill continuously and disabling any you don't trust.
