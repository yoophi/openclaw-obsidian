# Hermes vs OpenClaw 비교 분석

조사 일자: 2026-03-16

## 핵심 요약

두 가지 프로덕션급 오픈소스 AI 에이전트 비교:
- **OpenClaw**: TypeScript/Node.js 기반 통합 게이트웨이 플랫폼
- **Hermes**: Python 기반 자기 개선형 에이전트

---

## 1. 아키텍처 비교

### OpenClaw

| 특징 | 설명 |
|------|------|
| **기반** | TypeScript/Node.js |
| **구조** | 이벤트 스트리밍, 멀티채널 라우팅 |
| **도구 시스템** | Plugin SDK, phase-tracked |
| **핵심 철학** | 통합 플랫폼, 다중 채널/디바이스 |

### Hermes

| 특징 | 설명 |
|------|------|
| **기반** | Python |
| **구조** | 단일 에이전트 루프 |
| **도구 시스템** | Registry 기반, 병렬 실행 |
| **핵심 철학** | 학습과 투명성, 자기 개선 |

---

## 2. 상세 비교

### 2-1. 아키텍처

| 항목 | OpenClaw | Hermes |
|------|----------|--------|
| **기반 언어** | TypeScript/Node.js | Python |
| **구조** | 이벤트 스트리밍 | 단일 에이전트 루프 |
| **도구 호출** | Phase-tracked | 병렬 실행 |
| **확장성** | Plugin SDK | Registry 기반 |
| **프로바이더** | 광범위한 장애 조치 | Anthropic API, 프롬프트 캐싱 |

### 2-2. 메모리 & 비용

| 항목 | OpenClaw | Hermes |
|------|----------|--------|
| **세션 저장** | 세션별 트랜스크립트 | SQLite + 의미 검색 |
| **메모리 백엔드** | 파일/벡터 지원 | 계층형 (MD 파일 + DB) |
| **컨텍스트 압축** | 청크 요약 | protect-first/last 알고리즘 |
| **학습 능력** | 없음 | 자율 "스킬" 생성 |
| **비용 최적화** | 일반적 | 스킬 재사용 → 토큰 절감 |

**Hermes의 학습 특징:**
- 복잡한 문제 해결 후 자율적으로 스킬 생성
- 생성된 스킬은 편집 및 재사용 가능
- 반복 작업 효율 향상
- 프롬프트 캐시 재사용으로 비용 절감

### 2-3. 보안 모델

| 항목 | OpenClaw | Hermes |
|------|----------|--------|
| **신뢰 모델** | 개인 비서 모델 | 패턴 기반 탐지 |
| **인증** | 페어링 | - |
| **샌드박싱** | 선택적 | - |
| **위험 명령 탐지** | - | 패턴 기반 |
| **프롬프트 인젝션** | - | 탐지기 내장 |
| **메모리 쓰기** | - | 안전하지 않은 쓰기 차단 |
| **파일 스캔** | - | 스킬/컨텍스트 파일 스캔 |

---

## 3. 장단점

### OpenClaw

**장점:**
- 성숙하고 확장 가능한 플랫폼
- 멀티 채널/디바이스 통합
- 광범위한 프로바이더 지원
- Plugin SDK로 확장 용이

**단점:**
- Python 개발자에게 덜 익숙
- 학습/자기 개선 기능 없음

### Hermes

**장점:**
- Python 네이티브
- 투명하고 검사/확장 쉬움
- 자율 스킬 생성으로 학습
- 비용 효율적 (프롬프트 캐시 재사용)

**단점:**
- 단일 에이전트 중심
- 멀티 채널 통합 제한적
- 상대적으로 덜 성숙

## 4. 선택 가이드

| 상황 | 추천 |
|------|------|
| **통합 멀티 디바이스 비서** | OpenClaw |
| **Python 네이티브 개발** | Hermes |
| **투명하고 검사 쉬운** | Hermes |
| **성숙한 확장 플랫폼** | OpenClaw |
| **비용 효율성 중시** | Hermes |
| **학습/자기 개선 필요** | Hermes |
| **다중 채널 통합** | OpenClaw |

---

## 5. 결론

| 플랫폼 | 적합한 사용자 |
|--------|--------------|
| **OpenClaw** | 멀티 채널/디바이스 통합이 필요한 사용자, TypeScript 친화적 |
| **Hermes** | Python 개발자, 투명성과 비용 효율성 중시, 학습형 에이전트 선호 |
