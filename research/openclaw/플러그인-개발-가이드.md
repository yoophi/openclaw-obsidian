# OpenClaw 플러그인 개발 가이드

조사 일자: 2026-03-15

## 핵심 요약

OpenClaw 플러그인은 **TypeScript 모듈**로, Gateway와 동일한 프로세스에서 실행되며 다양한 기능을 확장할 수 있다. 도구, 채널, 명령어, HTTP 라우트, 백그라운드 서비스 등을 등록할 수 있다.

---

## 1. 플러그인 아키텍처

### 4개 레이어

1. **Manifest + Discovery** — `openclaw.plugin.json` + package metadata로 플러그인 발견
2. **Enablement + Validation** — 활성화/비활성화/차단 결정
3. **Runtime Loading** — jiti를 통해 in-process로 로드
4. **Surface Consumption** — Registry에서 도구/채널/명령어 등 노출

### 실행 모델

- **In-process**: Gateway와 동일한 프로세스에서 실행
- **신뢰 경계**: 플러그인은 코어 코드와 동일한 권한
- 보안: allowlist, 명시적 설치/로드 경로 사용 권장

## 2. 플러그인으로 할 수 있는 작업

### 2-1. Agent Tools 등록

AI 에이전트가 사용할 수 있는 도구 추가

```typescript
export default function (api) {
  api.registerTool({
    name: "my_tool",
    description: "이 도구는 ~을 합니다",
    parameters: {
      type: "object",
      properties: {
        query: { type: "string", description: "검색어" },
      },
      required: ["query"],
    },
    execute: async (params, context) => {
      // 도구 로직
      return { result: "..." };
    },
  });
}
```

### 2-2. Messaging Channel 등록

WhatsApp, Telegram 같은 메시징 채널 추가

```typescript
const myChannel = {
  id: "acmechat",
  meta: {
    id: "acmechat",
    label: "AcmeChat",
    selectionLabel: "AcmeChat (API)",
    docsPath: "/channels/acmechat",
    blurb: "데모 채널 플러그인",
    aliases: ["acme"],
  },
  capabilities: { chatTypes: ["direct", "group"] },
  config: {
    listAccountIds: (cfg) => Object.keys(cfg.channels?.acmechat?.accounts ?? {}),
    resolveAccount: (cfg, accountId) =>
      cfg.channels?.acmechat?.accounts?.[accountId ?? "default"],
  },
  outbound: {
    deliveryMode: "direct",
    sendText: async ({ text, to }) => {
      // 메시지 전송 로직
      return { ok: true };
    },
  },
};

export default function (api) {
  api.registerChannel({ plugin: myChannel });
}
```

### 2-3. Model Provider 등록

OAuth, API-key 인증을 통한 모델 프로바이더 추가

```typescript
api.registerProvider({
  id: "acme",
  label: "AcmeAI",
  auth: [
    {
      id: "oauth",
      label: "OAuth",
      kind: "oauth",
      run: async (ctx) => {
        // OAuth 플로우 실행
        return {
          profiles: [{ profileId: "acme:default", credential: {...} }],
          defaultModel: "acme/opus-1",
        };
      },
    },
  ],
  wizard: {
    onboarding: {
      choiceId: "acme",
      choiceLabel: "AcmeAI",
      methodId: "oauth",
    },
  },
  discovery: {
    order: "late",
    run: async () => ({
      provider: { baseUrl: "https://acme.example/v1", ... },
    }),
  },
});
```

### 2-4. Gateway HTTP Routes

플러그인 전용 HTTP 엔드포인트 추가

```typescript
api.registerHttpRoute({
  path: "/acme/webhook",
  auth: "plugin", // 또는 "gateway"
  match: "exact", // 또는 "prefix"
  handler: async (req, res) => {
    res.statusCode = 200;
    res.end("ok");
    return true; // 요청 처리 완료
  },
});
```

### 2-5. CLI Commands

`openclaw <command>` 명령어 추가

```typescript
api.registerCli(
  ({ program }) => {
    program.command("mycmd").action(() => {
      console.log("Hello from plugin");
    });
  },
  { commands: ["mycmd"] },
);
```

### 2-6. Auto-reply Commands (Slash Commands)

AI 없이 실행되는 슬래시 명령어

```typescript
api.registerCommand({
  name: "mystatus",
  description: "플러그인 상태 표시",
  acceptsArgs: true,
  requireAuth: true,
  handler: async (ctx) => {
    return { text: `상태: 실행 중\n인자: ${ctx.args}` };
  },
});
```

### 2-7. Gateway RPC Methods

Gateway RPC 메서드 등록

```typescript
api.registerGatewayMethod("myplugin.status", ({ respond }) => {
  respond(true, { ok: true, timestamp: Date.now() });
});
```

### 2-8. Background Services

백그라운드 서비스 등록

```typescript
api.registerService({
  id: "my-service",
  start: () => api.logger.info("서비스 시작"),
  stop: () => api.logger.info("서비스 중지"),
});
```

### 2-9. Lifecycle Hooks

에이전트 라이프사이클 훅 등록

```typescript
// 커맨드 훅
api.registerHook("command:new", async () => {
  // /new 실행 시 로직
}, { name: "my-plugin.command-new" });

// 라이프사이클 훅
api.on("before_prompt_build", (event, ctx) => {
  return {
    prependSystemContext: "회사 스타일 가이드를 따르세요.",
  };
}, { priority: 10 });
```

### 2-10. Context Engine

세션 컨텍스트 관리 교체

```typescript
api.registerContextEngine("my-engine", () => ({
  info: { id: "my-engine", name: "My Engine", ownsCompaction: true },
  async ingest() { return { ingested: true }; },
  async assemble({ messages }) { return { messages, estimatedTokens: 0 }; },
  async compact() { return { ok: true, compacted: false }; },
}));
```

## 3. 플러그인 개발 단계

### 3-1. 기본 구조

```
my-plugin/
├── openclaw.plugin.json  # 매니페스트
├── package.json          # npm 패키지 정보
├── index.ts              # 진입점
└── skills/               # 선택적 스킬
    └── my-skill/
        └── SKILL.md
```

### 3-2. Manifest 파일 (`openclaw.plugin.json`)

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "description": "플러그인 설명",
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": { "type": "string" },
      "enabled": { "type": "boolean" }
    }
  },
  "uiHints": {
    "apiKey": { "label": "API Key", "sensitive": true },
    "enabled": { "label": "활성화" }
  }
}
```

### 3-3. 진입점 (`index.ts`)

```typescript
// 함수 형태
export default function (api) {
  api.registerTool({...});
  api.registerCommand({...});
}

// 또는 객체 형태
export default {
  id: "my-plugin",
  name: "My Plugin",
  configSchema: {...},
  register(api) {
    api.registerTool({...});
  },
};
```

### 3-4. SDK Import

## 4. Discovery & Precedence

### 검색 순서

1. `plugins.load.paths` (config)
2. `<workspace>/.openclaw/extensions/*.ts`
3. `~/.openclaw/extensions/*.ts`
4. Bundled extensions (`<openclaw>/extensions/*`)

### 활성화 규칙

- `plugins.enabled: false` → 모든 플러그인 비활성화
- `plugins.deny` → 항상 우선
- `plugins.entries.<id>.enabled: false` → 해당 플러그인 비활성화
- 워크스페이스 플러그인 → 기본 비활성화
- `plugins.allow` → allowlist 적용

## 5. 설정

```json5
{
  plugins: {
    enabled: true,
    allow: ["voice-call"],
    deny: ["untrusted"],
    load: { paths: ["~/Projects/my-plugin"] },
    entries: {
      "voice-call": { enabled: true, config: { provider: "twilio" } },
    },
    slots: {
      memory: "memory-core",
      contextEngine: "legacy",
    },
  },
}
```

## 6. CLI 명령어

```bash
# 플러그인 목록
openclaw plugins list

# 설치
openclaw plugins install @openclaw/voice-call  # npm
openclaw plugins install ./my-plugin           # 로컬
openclaw plugins install -l ./my-plugin        # 링크 (개발용)

# 활성화/비활성화
openclaw plugins enable <id>
openclaw plugins disable <id>

# 업데이트
openclaw plugins update <id>
openclaw plugins update --all

# 진단
openclaw plugins doctor
```

## 7. 배포 (npm)

### 패키지 구조

```json
{
  "name": "@openclaw/my-plugin",
  "main": "dist/index.js",
  "openclaw": {
    "extensions": ["./dist/index.js"]
  }
}
```

### 설치 과정

1. `npm pack`으로 tarball 생성
2. `~/.openclaw/extensions/<id>/`에 추출
3. config에 자동 활성화

## 8. 보안 주의사항

- **In-process 실행**: 플러그인 버그가 Gateway 전체에 영향
- **신뢰**: 신뢰할 수 있는 플러그인만 설치
- **Allowlist**: `plugins.allow`로 명시적 허용
- **Shadowing**: 워크스페이스 플러그인이 bundled 플러그인을 덮어쓸 수 있음
