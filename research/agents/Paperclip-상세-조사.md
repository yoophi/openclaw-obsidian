# Paperclip 상세 조사

조사 일자: 2026-03-31

## 핵심 요약

Paperclip은 AI 에이전트 팀을 가상 회사처럼 운영할 수 있게 해주는 오픈소스 오케스트레이션 플랫폼이다. "OpenClaw가 직원이라면, Paperclip은 회사다"라는 컨셉으로, 다수의 AI 에이전트를 조직도, 예산, 거버넌스와 함께 관리한다.

---

## 1. 사용 사례

### 1.1 자율적 비즈니스 운영

**목표 설정 예시:**
> "Build the #1 AI note-taking app to $1M MRR"

- CEO 에이전트가 목표를 이슈로 분해
- Engineer 에이전트가 실행
- QA 에이전트가 품질 검토
- 대시보드에서 전체 진행 상황 모니터링

### 1.2 정기 작업 자동화

- **소셜 미디어 업데이트**: Heartbeat 스케줄로 자동 실행
- **고객 지원**: 정기적으로 미해결 티켓 확인
- **재무 보고**: 일일/주간 리포트 자동 생성
- **콘텐츠 생성**: 블로그, 뉴스레터 등 정기 생산

### 1.3 다중 비즈니스 운영

- 하나의 배포로 무제한 회사 운영 가능
- 완전한 데이터 격리
- 포트폴리오 비즈니스 관리에 적합

### 1.4 에이전트 감독

- 20개 이상의 Claude Code 터미널 추적
- 예산 초과 방지
- 작업 맥락 유지 (재부팅 후에도 지속)

## 2. OpenClaw와 연동 방법

### 2.1 관계 정의

| 구분 | OpenClaw | Paperclip |
|------|----------|-----------|
| 역할 | 개인 AI 에이전트 (직원) | 다중 에이전트 오케스트레이션 (회사) |
| 인터페이스 | 메시징 앱 (WhatsApp, Telegram, Discord) | 웹 대시보드 + 모바일 |
| 에이전트 지원 | 단일 에이전트 | 다중 에이전트 (OpenClaw, Claude Code, Codex, Cursor, Bash, HTTP) |
| 조직 구조 | 없음 | 조직도, 역할, 보고 체계 |
| 목표 정렬 | 없음 | 회사 미션까지 추적되는 작업 |
| 예산 관리 | 없음 | 에이전트별 월간 예산 + 자동 정지 |
| 감사 로그 | 기본 대화 기록 | 변경 불가능한 전체 도구 호출 추적 |
| 거버넌스 | 없음 | 채용/전략에 대한 이사회 승인 |
| 다중 회사 | 미지원 | 완전한 데이터 격리로 다중 회사 지원 |

### 2.2 연동 원리

**Heartbeat 인터페이스:**
- Paperclip은 heartbeat를 받을 수 있는 모든 에이전트와 연동 가능
- 어댑터가 Paperclip과 실행 환경 사이의 연결 처리
- 기존 에이전트를 재작성할 필요 없음

**지원되는 에이전트:**
- ✅ OpenClaw
- ✅ Claude Code
- ✅ Codex
- ✅ Cursor
- ✅ Bash 스크립트
- ✅ HTTP로 접근 가능한 모든 에이전트

### 2.3 연동 설정

```bash
# Paperclip 설치
npx paperclipai onboard --yes

# 또는 수동 설치
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

**설정 단계:**
1. 대시보드에서 회사 생성
2. 미션 스테이트먼트 작성
3. 에이전트 추가 (역할, 보고 체계, 직무 기술서)
4. Heartbeat 빈도 설정
5. 월간 토큰 예산 할당
6. 프로젝트 및 태스크 생성

## 3. 다른 AI 서비스 대비 특징

### 3.1 vs Agent Zero

| 항목 | Paperclip | Agent Zero |
|------|-----------|------------|
| 아키텍처 | 회사 모델 | Docker 샌드박스 하위 에이전트 |
| 브라우저 자동화 | 별도 구성 필요 | 기본 내장 |
| 복잡성 | 낮음 (시작하기 쉬움) | 높음 (Docker 필요) |
| 대시보드 | React UI 기본 제공 | CLI 중심 |
| 성숙도 | 초기 단계 | 더 성숙함 |

### 3.2 vs CrewAI

| 항목 | Paperclip | CrewAI |
|------|-----------|--------|
| 언어 | JavaScript/TypeScript | Python-first |
| 구성 방식 | 회사 구조로 추상화 | 코드로 역할/태스크 정의 |
| 대시보드 | 기본 제공 (localhost:3100) | 별도 구축 필요 |
| 학습 곡선 | 낮음 | 보통 |
| 커스터마이징 | 템플릿 기반 | 코드 레벨 제어 |

### 3.3 vs AutoGen

| 항목 | Paperclip | AutoGen |
|------|-----------|---------|
| 개발사 | 커뮤니티 | Microsoft |
| 방향성 | 실용적 로컬 자동화 | 연구 중심 |
| 복잡한 대화 패턴 | 제한적 | 강력함 |
| 생태계 | 초기 단계 | 성숙함 |
| 진입 장벽 | 낮음 | 높음 |

### 3.4 Paperclip의 고유 특징

**장점:**
- ✅ 친숙한 비유: 회사/프로젝트/이슈 모델
- ✅ 즉시 사용 가능한 대시보드
- ✅ 내장 Postgres (별도 DB 설정 불필요)
- ✅ 예산 추적 및 자동 정지
- ✅ 16개 사전 구축된 회사 템플릿
- ✅ MIT 라이선스, 완전한 셀프 호스팅

**단점:**
- ❌ 초기 단계 프로젝트 (API 변경 가능성)
- ❌ 셀프 호스팅 전용 (클라우드 버전 없음)
- ❌ 제한된 커뮤니티 리소스
- ❌ 단일 머신 범위 (분산 배포 미지원)
- ❌ Claude 어댑터가 가장 성숙, 다른 모델 지원은 진행 중

## 4. 아키텍처

### 4.1 기술 스택

- **백엔드**: Node.js 서버
- **프론트엔드**: React UI
- **데이터베이스**: PostgreSQL (내장)
- **포트**: 3100 (기본)

### 4.2 핵심 개념

| 개념 | 설명 |
|------|------|
| Company | 최상위 조직 단위 |
| Goal | 회사의 미션 스테이트먼트 |
| Project | 골 하위의 작업 그룹 |
| Issue | 개별 태스크 (티켓) |
| Role | CEO, Engineer, QA 등 |
| Heartbeat | 정기 실행 스케줄 |
| Budget | 에이전트별 월간 토큰 예산 |
| Governance | 이사회 승인 프로세스 |

### 4.3 실행 모델

```
Goal (회사 미션)
  └── Project (프로젝트)
      └── Issue (이슈/태스크)
          └── Agent (에이전트가 실행)
              └── Tool Calls (도구 호출 - 모두 로깅됨)
```
